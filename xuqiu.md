接下来我们要讨论的是，本地 agent 和服务端 agent 如何在这个软件中融合、和谐共生。

我遵循的是一种 Local-first（本地优先）的模式。也就是说，当用户刚进来登录之后，开始聊天时，使用的就是一个本地 agent；当你需要使用远程 agent 的时候，就在服务端上创建一个远程 agent。

关于这个理念，目前的 Client 里有一个 Workspace（工作区）的概念，一个人可以同时创建多个 Workspace，其实每个 Workspace 就相当于一个本地智能体。但问题在于，左侧菜单栏里的“智能体”项，列出的是该账号下所有的智能体，既包括服务端的，也包括本地的。

从产品设计的角度来看，我认为这块需要统一：

如果沿用“工作区”这个概念，那么所有相关地方都要用工作区；
如果用的是 Agent（智能体）的概念，那就统一改成“智能体”字样。
